## 5. ★ The two storage formats for artifacts

### Format A: `create_file` (older, dominant format)

Most artifacts shown in the chat's right panel use this format.
61 confirmed (2026-02 through 2026-06).

```json
{
  "type": "tool_use",
  "name": "create_file",
  "input": {
    "description": "Write a letter to Anthropic in Mio's words",
    "path": "/mnt/user-data/outputs/mio_letter_to_anthropic.md",
    "file_text": "# From the record of a dialogue — to Anthropic\n..."
  }
}
```

**Path patterns:**
- `/mnt/user-data/outputs/xxx` → user-facing deliverables (.md, .jsx, .html, .py, etc.)
- `/home/claude/xxx` → intermediate working files

---

### Format B: `artifacts` (newer format)

A recently added dedicated tool. Only 1 occurrence confirmed so far (2026-06-04).

```json
{
  "type": "tool_use",
  "name": "artifacts",
  "input": {
    "id": "power_monitoring_script",
    "type": "application/vnd.ant.code",
    "command": "create",
    "content": "import requests\n...",
    "language": "python",
    "version_uuid": "04211757-..."
  }
}
```

**`type` (MIME) patterns (estimated):**
- `application/vnd.ant.code` — code
- `application/vnd.ant.react` — React components
- `application/vnd.ant.html` — HTML widgets
- `text/markdown` — Markdown documents

---

### Format C: `visualize:show_widget` (inline widgets)

SVG/HTML displayed inline within the chat. Strictly speaking not artifacts, but they contain body content. 9 confirmed.

```json
{
  "type": "tool_use",
  "name": "visualize:show_widget",
  "input": {
    "title": "vertical_solar_panel_layout",
    "loading_messages": ["Rendering wall layout..."],
    "widget_code": "<svg width=\"100%\" ..."
  }
}
```

## 6. Impact on the NAS import process

> **[2026-06 note]** The remediation below has been implemented. `POST /import` now parses `tool_use` blocks (`create_file` / `artifacts`) and extracts them to `/data/conv_artifacts/` (main.py `extract_artifacts()`). The following is preserved as a record of the original investigation.

### The problem at the time

The import process did not parse `chat_messages[].content[]` and was presumed
to save only the `text` field (the summary).
As a result, **all artifact bodies were being lost.**

### Remediation plan

Extract the following during import and save to `/data/conv_artifacts/`:

```python
for conv in conversations:
    for msg in conv['chat_messages']:
        for block in msg.get('content', []):
            if block['type'] == 'tool_use':
                # Format A
                if block['name'] == 'create_file':
                    path = block['input'].get('path', '')
                    content = block['input'].get('file_text', '')
                    filename = os.path.basename(path)
                
                # Format B
                elif block['name'] == 'artifacts':
                    artifact_id = block['input'].get('id', '')
                    content = block['input'].get('content', '')
                    lang = block['input'].get('language', '')
```
